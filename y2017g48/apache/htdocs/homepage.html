Homework 1.2:In a normal intercommunication between Web browser and Web Server, Web Server closes the network communication right after it sends back the document requested by the browser. Think about this statusless character of World Wide Web. Give some good points and bad points of it. Can a web server know exactly how many browsers are reading its document?

Good Point:
After transmitting the document, the web server immediately closes the TCP, which can make server have more time to deal with other browsers' request, Moreover, Server TCP has the maximum number of connections, if not closed, too much visitor will lead to the server tcp port blocked.
Bad Point:
If we are browsering many websides which are very similiar with each other at the same time, the connecting repeatedly will also waste much time and transmite a lot of the same data, it's the bad point I can considered.
I think the server unable to know how many browsers are reading its document, because the server closes the TCP after transmitting the document data, which means that serve will not know when the browser closes the document. So I think the server unable to know how many browsers are reading the document.
